## ADDED Requirements

### Requirement: 本地实例管理边界
实例管理 SHALL 仅管理当前私有化部署实例的初始化状态、允许的系统配置、账号启停、审计查询以及版本与健康信息。该能力 MUST NOT 提供家庭目录、家庭数量与注册趋势等运营统计、租户控制面或厂商云依赖；家庭业务数据只能由具备 `PARENT` 角色的用户通过家庭能力访问。
#### Scenario: 查看实例管理概览

- **WHEN** 有效 `INSTANCE_ADMIN` 请求实例管理概览
- **THEN** 系统仅返回初始化状态、脱敏配置摘要、账号状态摘要、审计入口、版本、健康信息、最近备份时间与状态、下次计划备份时间以及最近恢复演练结果
- **AND** 响应不包含家庭成员详情、孩子资料、任务、积分、奖励或兑换数据，也不包含备份路径或凭据

#### Scenario: 请求家庭运营管理能力
- **WHEN** `INSTANCE_ADMIN` 请求家庭目录、家庭运营统计或租户级管理操作
- **THEN** 系统拒绝请求并返回 `CAPABILITY_NOT_SUPPORTED`

#### Scenario: 实例离线运行
- **WHEN** 当前私有化实例无法连接任何厂商云服务
- **THEN** 初始化状态、密码登录、实例配置、账号启停、审计查询、版本和本地健康检查 SHALL 仍可工作

### Requirement: 实例初始化状态
系统 SHALL 维护 `UNINITIALIZED` 与 `INITIALIZED` 两种实例初始化状态。未认证状态查询 MUST 仅返回完成首次设置所需的最小状态，且 MUST NOT 返回初始化令牌、账号资料、配置秘密或内部诊断信息；初始化一旦成功，常规 API MUST NOT 将实例恢复为未初始化状态。

#### Scenario: 查询未初始化实例
- **WHEN** 客户端查询尚未完成首次设置的实例状态
- **THEN** 系统返回 `UNINITIALIZED` 和启动初始化界面所需的最小非秘密信息
- **AND** 响应不包含一次性初始化令牌或其派生值

#### Scenario: 查询已初始化实例
- **WHEN** 客户端查询已经完成首次设置的实例状态
- **THEN** 系统返回 `INITIALIZED`，且不返回首位用户手机号、角色详情或任何秘密

#### Scenario: 初始化成功后尝试重置状态
- **WHEN** 客户端通过常规 API 请求将已初始化实例改回 `UNINITIALIZED`
- **THEN** 系统拒绝请求并返回 `INITIALIZATION_NOT_ALLOWED`

### Requirement: 实例管理员角色隔离
只有状态有效且具备 `INSTANCE_ADMIN` 角色的账号 SHALL 能执行实例管理操作。首位用户 MAY 同时具有 `INSTANCE_ADMIN` 与 `PARENT`，但系统 MUST 对实例管理与家庭业务分别执行角色检查，不得因角色共存而合并权限。

#### Scenario: 首位用户访问实例配置
- **WHEN** 同时具有 `INSTANCE_ADMIN` 与 `PARENT` 的首位用户请求实例配置
- **THEN** 系统基于 `INSTANCE_ADMIN` 角色授权，并仅返回实例配置允许字段

#### Scenario: 普通家长访问实例管理
- **WHEN** 仅具有 `PARENT` 角色的有效用户请求配置、账号启停、审计查询或详细健康信息
- **THEN** 系统拒绝请求并返回 `FORBIDDEN`

#### Scenario: 仅有实例管理员角色时访问家庭数据
- **WHEN** 仅具有 `INSTANCE_ADMIN` 角色的用户请求家庭成员或家庭业务数据
- **THEN** 系统拒绝请求并返回 `FORBIDDEN`

### Requirement: 系统配置管理
有效 `INSTANCE_ADMIN` SHALL 能查看和更新明确列入白名单的本地系统配置。系统 MUST 校验配置类型、范围与组合约束，并 MUST 将服务商密钥、签名密钥、数据库凭据及其他秘密字段设为只写或脱敏显示；配置变更 MUST 原子应用并产生审计事件。

#### Scenario: 查看系统配置
- **WHEN** 有效 `INSTANCE_ADMIN` 请求系统配置
- **THEN** 系统返回允许配置项的当前非秘密值及秘密是否已配置的状态
- **AND** 秘密字段仅显示固定掩码或“已配置”，不返回明文、密文或可逆片段

#### Scenario: 更新有效配置
- **WHEN** 有效 `INSTANCE_ADMIN` 提交通过校验的白名单配置变更
- **THEN** 系统原子保存并按配置约定生效
- **AND** 系统记录操作者、时间、结果和变更字段名，但不记录秘密的新旧值

#### Scenario: 更新无效或未知配置
- **WHEN** 请求包含未知配置项、无效类型、越界值或不完整的关联配置
- **THEN** 系统拒绝整个更新并返回 `CONFIGURATION_INVALID`
- **AND** 所有现有配置保持不变

#### Scenario: 短信配置不完整
- **WHEN** 短信服务商所需配置缺失、无效或未显式启用
- **THEN** 系统将短信登录能力保持为关闭状态
- **AND** 手机号加密码登录及本地管理员恢复不受影响

### Requirement: 当前实例账号启停
有效 `INSTANCE_ADMIN` SHALL 能按页查看当前实例账号的最小化状态，并能启用或停用账号。列表和操作响应 MUST 对手机号等敏感字段脱敏，且 MUST NOT 返回凭据、会话、家庭业务详情或孩子隐私资料。停用操作 MUST 原子撤销目标账号全部会话；重新启用 MUST NOT 恢复旧会话。

#### Scenario: 查看账号状态列表
- **WHEN** 有效 `INSTANCE_ADMIN` 请求当前实例账号状态列表
- **THEN** 系统返回分页的非秘密账号标识、脱敏手机号、角色、启停状态和必要时间信息
- **AND** 响应不包含孩子资料、家庭业务数据、密码哈希、PIN 哈希或令牌

#### Scenario: 停用普通账号
- **WHEN** 有效 `INSTANCE_ADMIN` 停用一个不会破坏最后角色约束的有效账号
- **THEN** 系统将账号标记为停用并立即撤销该账号的全部会话
- **AND** 系统记录账号停用审计事件

#### Scenario: 重新启用账号
- **WHEN** 有效 `INSTANCE_ADMIN` 重新启用一个已停用账号
- **THEN** 系统允许该账号使用现有有效凭据重新认证
- **AND** 系统不恢复停用前的任何访问令牌、刷新令牌或孩子设备会话

#### Scenario: 保护最后一位实例管理员
- **WHEN** 停用、删除或移除角色将导致实例不再有可登录的 `INSTANCE_ADMIN`
- **THEN** 系统拒绝整个操作并返回 `LAST_INSTANCE_ADMIN`
- **AND** 目标账号、角色和现有会话保持不变

#### Scenario: 保护最后一位有效家长
- **WHEN** 账号停用将导致唯一家庭不再有可登录的有效 `PARENT`
- **THEN** 系统拒绝整个操作并返回 `LAST_ACTIVE_PARENT`
- **AND** 目标账号和现有会话保持不变

#### Scenario: 并发停用受保护账号
- **WHEN** 多个账号状态请求并发尝试停用剩余的管理员或家长
- **THEN** 系统 SHALL 以原子约束确保至少一位可登录的 `INSTANCE_ADMIN` 和一位可登录的 `PARENT` 保留
- **AND** 违反约束的请求返回相应稳定错误码

#### Scenario: 操作非当前实例账号
- **WHEN** 请求使用不存在或不属于当前实例的账号标识
- **THEN** 系统返回不泄露账号存在性的 `RESOURCE_NOT_FOUND`

### Requirement: 敏感操作审计记录
系统 MUST 为初始化与恢复、登录成功与失败、设备绑定与撤销、邀请和成员变更、孩子档案与 PIN 变更、系统配置、账号启停及数据导出记录审计事件。每条事件 SHALL 包含操作者或系统来源、发生时间、动作类型、结果、关联对象的非秘密标识和请求关联标识，并 MUST 屏蔽密码、PIN、验证码、令牌、Cookie、邀请秘密、设备凭据、配置秘密及不必要的个人信息。

#### Scenario: 记录成功的敏感操作
- **WHEN** 用户、系统任务或本地维护流程成功执行受审计操作
- **THEN** 系统写入包含操作者、时间、结果和关联对象的审计事件
- **AND** 事件中的手机号、网络地址或其他敏感标识按查询目的最小化或脱敏

#### Scenario: 记录失败的敏感操作
- **WHEN** 登录、授权、配置、成员、绑定、账号或恢复操作因校验、权限或安全限制失败
- **THEN** 系统记录失败结果和稳定原因码
- **AND** 事件不记录请求中的秘密字段或原始凭据

#### Scenario: 应用用户尝试修改审计记录
- **WHEN** 任一应用用户请求更新或删除既有审计事件
- **THEN** 系统拒绝请求并返回 `AUDIT_IMMUTABLE`

### Requirement: 审计日志查询
有效 `INSTANCE_ADMIN` SHALL 能按时间范围、动作类型、结果、操作者非秘密标识和关联对象非秘密标识查询当前实例审计日志。查询 MUST 分页、受授权检查和资源限制保护，并 MUST 在返回前再次执行字段脱敏。

#### Scenario: 使用允许条件查询审计日志
- **WHEN** 有效 `INSTANCE_ADMIN` 提交合法时间范围和允许的筛选条件
- **THEN** 系统返回当前实例内匹配的分页审计事件及分页游标
- **AND** 每条事件仅包含审计目的所需的脱敏字段

#### Scenario: 查询范围过大
- **WHEN** 审计查询超过配置的时间范围、页大小或请求频率限制
- **THEN** 系统拒绝或收窄请求，并返回 `AUDIT_QUERY_LIMIT_EXCEEDED` 或 `RATE_LIMITED`

#### Scenario: 普通家长查询审计日志
- **WHEN** 不具有 `INSTANCE_ADMIN` 角色的用户请求实例审计日志
- **THEN** 系统拒绝请求并返回 `FORBIDDEN`

### Requirement: 版本与健康信息
系统 SHALL 提供本地版本、存活状态、就绪状态、备份运行状态与恢复演练结果。未认证健康接口 MUST 仅暴露探针所需的最小布尔状态；详细版本、构建、本地依赖健康信息、最近备份时间与状态、下次计划备份时间以及最近恢复演练结果 MUST 仅向有效 `INSTANCE_ADMIN` 提供，并 MUST 排除秘密、个人信息、内部堆栈和可被利用的连接详情。

#### Scenario: 未认证存活探针
- **WHEN** 本地编排或监控请求未认证的存活接口
- **THEN** 系统仅返回可机器判断的存活状态和适当 HTTP 状态
- **AND** 响应不包含版本、账号、配置或依赖连接信息
#### Scenario: 管理员查看详细健康信息

- **WHEN** 有效 `INSTANCE_ADMIN` 请求详细版本与健康信息
- **THEN** 系统返回应用版本、构建标识、数据库迁移状态、允许的本地依赖健康摘要、最近备份时间与状态、下次计划备份时间以及最近恢复演练结果
- **AND** 响应不包含数据库凭据、服务商密钥、内部堆栈、备份路径或个人信息

#### Scenario: 恢复演练结果可核查

- **WHEN** 有效 `INSTANCE_ADMIN` 请求实例管理概览或详细健康信息
- **THEN** 系统返回最近一次恢复演练的时间、成功状态、RPO 与 RTO 实际值，以及当当前实际 RPO 超过 24 小时时返回 `RPO_EXCEEDED` 警告状态

#### Scenario: 本地依赖未就绪
- **WHEN** 数据库或必需本地依赖不可用
- **THEN** 就绪接口返回未就绪状态和稳定原因码
- **AND** 未认证响应不披露依赖地址、凭据或异常堆栈

### Requirement: 首位管理员本地凭据恢复
部署方 SHALL 能在实例主机上通过本地维护命令发起首位管理员的一次性、限时凭据恢复流程。恢复 MUST 依赖本地主机授权而非厂商云服务，MUST 不创建新账号、不新增角色、不改变家庭归属，并 MUST 在成功、过期或撤销后使恢复凭据失效。

#### Scenario: 从实例主机发起恢复
- **WHEN** 获得实例主机维护权限的部署方执行恢复命令
- **THEN** 系统为首位管理员生成一个限时且单次使用的恢复流程
- **AND** 恢复秘密仅输出到受控本地交互上下文，不写入应用日志或审计详情

#### Scenario: 完成一次性凭据恢复
- **WHEN** 部署方在有效期内使用恢复流程为首位管理员设置符合策略的新密码
- **THEN** 系统替换该账号密码哈希、撤销其全部现有会话并消费恢复凭据
- **AND** 系统保留原账号标识、`INSTANCE_ADMIN` 与 `PARENT` 角色及唯一家庭归属

#### Scenario: 重用或使用过期恢复凭据
- **WHEN** 客户端提交已使用、已撤销、已过期或无效的恢复凭据
- **THEN** 系统拒绝请求并返回 `RECOVERY_NOT_AVAILABLE`
- **AND** 系统不披露首位管理员手机号、账号状态或恢复流程细节

#### Scenario: 非本地主机发起恢复
- **WHEN** 远程 API 客户端尝试创建首位管理员恢复流程
- **THEN** 系统拒绝请求并返回 `FORBIDDEN`
- **AND** 系统不调用外部厂商服务或生成恢复凭据

### Requirement: 实例管理防滥用与错误语义
系统 MUST 对未认证状态查询、初始化提交、恢复凭据校验和高成本审计查询实施可配置限流。实例管理 API SHALL 使用稳定错误码与一致响应结构，并 MUST 在授权失败、对象不存在或内部错误时屏蔽账号存在性、秘密字段和内部实现详情。

#### Scenario: 初始化或恢复请求超过限制
- **WHEN** 同一来源或关联标识在限制窗口内超过初始化或恢复请求阈值
- **THEN** 系统返回 HTTP 429 和 `RATE_LIMITED`
- **AND** 响应不披露初始化令牌、恢复凭据或首位管理员状态

#### Scenario: 实例管理发生内部错误
- **WHEN** 实例管理操作因未处理的内部异常失败
- **THEN** 系统返回通用稳定错误码和请求关联标识
- **AND** 响应不包含堆栈、SQL、配置值、文件路径、手机号或其他敏感字段
